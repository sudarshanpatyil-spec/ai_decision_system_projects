Nightly Orchestration Engine

1
Night Job Pipeline
Runs daily at 02:00 AM — processes entire user base and updates campaign cohorts
02:00 AM — Step 1
Feature Store Snapshot
Pull latest behavioural signals (login, trades, sessions, notifications) and profile data for all eligible users from the data warehouse. Approx 5M users processed.
02:30 AM — Step 2
XGBoost Batch Prediction
For each user, run all 3 models (activation / reactivation / retention) across all 36 combos. Outputs P(convert|no_campaign) and P(convert|combo[i]) for each user × combo pair.
04:00 AM — Step 3
Prediction Table Creation
Compute Incremental Business Value for all combos. Rank top 10 combos per user per journey type. Write to PREDICTION_TABLE partitioned by date and user_id.
04:30 AM — Step 4
Orchestration Layer
Apply ε-greedy (90/10 exploitation/exploration), contact frequency policies, compliance blocks, and segment eligibility rules. Assign final campaign for each user.
05:00 AM — Step 5
CleverTap Cohort Update
12 campaign cohorts (4 activation + 4 reactivation + 4 retention, one per segment) pushed to CleverTap via API. Campaigns scheduled for optimal send time per user.

Prediction Table Structure
Top 10 combos per user ranked by Incremental Business Value (IBV)
Only the top 10 entries per user are retained in the prediction table and fed to the orchestration layer. These become the candidate pool for ε-greedy selection. Row #1 is the "exploit" choice (90% probability); rows #2–10 are "explore" candidates (10% probability, weighted by IBV rank).

Why exploration matters: Without it, the model would always choose the same combo and never learn if alternatives perform better. The 10% explore budget — weighted so Rank 2 beats Rank 10 — collects user feedback that continuously retrains the model and prevents exploitation bias.
ε-Greedy Decision Logic (Per User)
function selectCampaign(user_id, top10) {
  // Draw random uniform [0, 1]
  const rand = Math.random();

  if (rand < 0.90) {
    // EXPLOIT: best combo (rank 1)
    return top10[0];
  } else {
    // EXPLORE: weighted sample [1..9]
    const weights = [0.30, 0.22, 0.17, 0.13,
                      0.07, 0.04, 0.03, 0.02, 0.02];
    return weightedSample(top10.slice(1), weights);
  }
}
Feedback loop → model improvement
User responds (or not) → feedback written to event store → next training cycle uses this data → model learns which combo truly works for which user type.
4
Contact Policies & Guardrails
Orchestration layer applies these checks before campaign assignment
📅
Daily Frequency Cap
User cannot receive more than 1 campaign per day across all channels and segments.
MAX 1/DAY
📆
Weekly Frequency Cap
No more than 3 campaigns per week. Prevents fatigue and ensures unsubscribe rate stays low.
MAX 3/WEEK
🚫
Compliance Segment Block
If user is compliance-blocked for a specific segment (FNO/Cash/MF/Commodity), that segment's campaign is suppressed.
SEGMENT CHECK
⏰
Optimal Send Time
Message delivered in user's historically highest open-rate time window (personalised per user).
PERSONALISED
🔒
DND / Opt-Out Check
Users on Do Not Disturb or who opted out of a channel are excluded from that channel's campaigns.
HARD BLOCK
🔁
Cooldown After Conversion
If user converted (activated/reactivated/retained) in last 7 days, suppress further campaign to avoid spam.
7-DAY COOLDOWN
